Understanding git diff

1. What is git diff?

The git diff command shows the changes between commits, branches, files, and more. It is commonly used to see what has changed in your working directory compared to the staging area or a commit.

2. Common Uses

  • Compare Working Directory with Staging Area: git diff
  • Compare Staging Area with Last Commit: git diff --staged
  • Compare Two Commits: git diff commit1 commit2
  • Compare Two Branches: git diff branch1 branch2
  • Check Changes in a Specific File: git diff file.txt

3. Example Output

diff --git a/index.html b/index.html
index e69de29..4b825dc 100644
--- a/index.html
+++ b/index.html
@@ -0,0 +1,3 @@
+<h1>Hello World</h1>
+<p>This is a new line</p>

Legend:
- Lines removed
+ Lines added

4. Tips

  • Use git diff --color for colored output in the terminal.
  • Use git difftool to view changes in a visual diff tool.
  • Remember: git diff by default does NOT show staged changes unless you add --staged.
