Github Actions

1. What is GitHub Actions?

GitHub Actions is a CI/CD (Continuous Integration/Continuous Deployment) platform that allows you to automate your build, test, and deployment pipeline directly from your GitHub repository.

  1. Built in CI/CD tool in github.
  2. Automate tasks like testing, building, deploying.
  3. Works like a virtual assistant for developers.

2. Why do we need Github Actions?

GitHub Actions helps automate workflows, ensuring that code changes are tested and deployed efficiently.

  • Automate builds and tests.
  • Improves code quality.
  • Auto deployments.
  • Scalable and Secure.

3. Github Actions Workflow?

  • Automated process that runs based on triggers
  • Defined using YAML files inside the .github/workflows folder

4. Key Components of a workflow?

  • Event: Triggers that start the workflow (e.g., push, pull request).
  • Job: A set of steps that execute on a specific runner.
  • Step: Individual tasks within a job (e.g., checkout code, run tests).
  • Runner: The environment where the job is executed (e.g., GitHub-hosted, self-hosted).

Workflow Execution Flow

 Event Occurs -> Workflow Starts -> Job execute in parallel(by default)

How to execute workflow on event trigger

repo-name/.github/workflows/main.yml 

    name: Harshal Khairnar

    on:
        push:
          branches: [main]

    jobs:
        demo:
          runs-on: ubuntu-latest

        steps:
            - name: Greetings
            run: echo "Hello From Harshal"

How to Schedule Github Actions Workflows

repo-name/.github/workflows/main.yml 

    name: Schedule Job

    on:
        schedule:
            - cron: '0 0 * * *'   # 12am-everyday       # minute, hour, date, month, week   # cron-> command run on

    jobs:
        run-script:
           runs-on: ubuntu-latest

        steps:
            - name: Print Message
            run: echo "Schedule Workflow Executed"

How to Disable Github Actions Workflows

  1. Go to your repository on GitHub.
  2. Click on the "Actions" tab.
  3. Select the workflow you want to disable.
  4. Click on the "..." button (three dots) in the top right corner.
  5. Select "Disable workflow" from the dropdown menu.

React Build Workflow

You can automate your React build process using GitHub Actions so that every time you push code (or merge PR), it automatically installs dependencies, builds the project, and even deploys (if you want).

name: React Build CI

on:
  push:
    branches:
      - main   # run only when pushing to main branch
  pull_request:
    branches:
      - main   # also run on PRs targeting main

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      # Step 1: Checkout repo
      - name: Checkout code
        uses: actions/checkout@v4

      # Step 2: Setup Node.js
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '22'  
          cache: 'npm'

      # Step 3: Install dependencies
      - name: Install dependencies
        run: npm install

      # Step 4: Run tests (optional)
      - name: Run tests
        run: npm test -- --watchAll=false

      # Step 5: Build React App
      - name: Build React App
        run: npm run build

      # Step 6: Upload build artifacts (optional)
      - name: Upload production-ready build
        uses: actions/upload-artifact@v4
        with:
          name: react-build
          path: build

Artifacts and Caching for Next.js Build

Caching

Caching stores dependencies or build files for future workflow runs.
Example: node_modules or .next/cache

Why use Caching?

  • Speeds up builds by reusing previous dependencies.
  • Reduces bandwidth (no need to re-download all npm/yarn packages).
  • Makes CI/CD faster and cheaper.

Need in Next.js

Next.js uses .next/cache to store precompiled pages and webpack data.
Caching this between runs avoids rebuilding everything from scratch → faster builds.

Artifacts

Artifacts are files generated during the workflow (e.g., build outputs, logs, test results) that can be stored and downloaded later.
In Next.js, the .next folder (build output) can be stored as an artifact.

Why use Artifacts?

  • Share build outputs between jobs in the same workflow (e.g., build in one job, test/deploy in another).
  • Debugging: save logs, screenshots, or reports.
  • Avoid rebuilding in later stages (saves CI/CD time).

Need in Next.js

Next.js builds take time. If you store the .next folder as an artifact in one job, you don’t need to rebuild it again for deployment.

name: Next JS Build with Caching and Artifacts

on:
  push:
    branches: [main]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      # Step 1: Checkout repository
      - name: Checkout Repository
        uses: actions/checkout@v4.2.2

      # Step 2: Setup Node.js
      - name: Setup Node JS
        uses: actions/setup-node@v4.2.0
        with:
          node-version: 22.x  
          cache: 'npm'

      # Step 3: Install dependencies
      - name: Install dependencies
        run: npm install

      # Step 4: Build Next JS App
      - name: Build Next JS App
        run: npm run build

      # Step 5: Upload build on Artifacts 
      - name: Upload Built on Artifacts
        uses: actions/upload-artifact@v4.6.0
        with:
          name: nextjs-build
          path: out/
