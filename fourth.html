What is .gitignore?

The .gitignore file tells Git which files or folders to ignore. Ignored files are not tracked, committed, or pushed to the remote repository.

Why use it?

  • Prevent sensitive data (e.g., passwords, API keys) from being uploaded.
  • Ignore large or unnecessary files like node_modules or build outputs.
  • Keep the repository clean and focused on source code.

Common .gitignore Rules

1. Ignore a specific file

secret.txt

2. Ignore a specific folder

node_modules/
build/
dist/

3. Ignore all files with a specific extension

*.log    # all log files
*.tmp    # all temporary files
*.zip    # all zip files

4. Ignore files in any folder matching a pattern

**/*.log   # any .log file in any subfolder
**/*.class # Java compiled files anywhere

5. Ignore multiple items at once

# Folders
node_modules/
build/

# Files
secret.txt
config.json

# Extensions
*.log
*.tmp

6. Negate a pattern (force track a file)

Use ! before the file name to keep it tracked even if its pattern is ignored. 

*.log
!important.log

Important Notes

  • If a file is already tracked by Git, adding it to .gitignore won’t stop tracking it.
  • To untrack a file: 
    git rm --cached filename
  • .gitignore should be committed so everyone working on the repo ignores the same files.

Example Project Structure

project/
  ├── index.js
  ├── package.json
  ├── node_modules/     # Ignored
  ├── .env              # Ignored
  ├── error.log         # Ignored
  └── .gitignore
