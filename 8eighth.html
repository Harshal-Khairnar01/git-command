Creating Git Aliases

Git aliases let you create shortcuts for frequently used Git commands. You can set them globally (for all repositories) or locally (for one repository only).

1. Global Aliases

These apply to all Git repositories on your system.

# Syntax
git config --global alias.<shortcut> '<command>'

# Examples
git config --global alias.st status
git config --global alias.co checkout
git config --global alias.br branch
git config --global alias.cm 'commit -m'

After setting these, you can use:
git st instead of git status
git co main instead of git checkout main

2. Local Aliases

These apply only to the repository where you set them.

# Navigate to your repository
cd my-project

# Syntax
git config alias.<shortcut> '<command>'

# Example
git config alias.last 'log -1 HEAD'

Now git last will show the latest commit in that repository only.

3. View Existing Aliases

# Global aliases
git config --global --list | grep alias

# Local aliases
git config --list | grep alias

4. Remove an Alias

# Global
git config --global --unset alias.st

# Local
git config --unset alias.last

Tips

  • Aliases cannot override built-in Git commands.
  • For complex commands, wrap them in quotes, e.g.: 
    git config --global alias.hist "log --oneline --graph --decorate --all"
  • Aliases are stored in:
    • Global: ~/.gitconfig
    • Local: .git/config inside the repository
