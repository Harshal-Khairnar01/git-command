GitHub Guide: Creating & Merging Pull Requests

1. Fork & Clone Repository

  1. Open the repository you want to contribute to on GitHub.
  2. Click Fork (top right corner) to copy it to your account.
  3. Clone your forked repo to your local machine: 
    git clone https://github.com/your-username/repo-name.git

2. Create a New Branch

  1. Navigate to your project folder: 
    cd repo-name
  2. Create a new branch for your feature or fix: 
    git checkout -b feature-branch-name

3. Make Changes & Commit

  1. Edit files and make your changes.
  2. Stage the changes: 
    git add .
  3. Commit the changes: 
    git commit -m "Add your commit message here"

4. Push Changes to GitHub

  1. Push your branch to your GitHub fork: 
    git push origin feature-branch-name

5. Create Pull Request (PR)

  1. Go to the original repository on GitHub.
  2. Click Pull Requests â†’ New Pull Request.
  3. Select your branch from the compare dropdown.
  4. Write a clear title and description for your changes.
  5. Click Create Pull Request.

6. Review & Merge Pull Request

  1. The repository maintainer will review your changes.
  2. If there are requested changes, update your branch and push again: 
    git add .  
git commit -m "Address review feedback"  
git push origin feature-branch-name
  3. Once approved, the PR can be merged using:
    • Merge Pull Request (keeps history)
    • Squash & Merge (combines commits)
    • Rebase & Merge (linear history)

7. Clean Up Branch

  1. Delete the branch locally: 
    git branch -d feature-branch-name
  2. Delete the branch on GitHub: 
    git push origin --delete feature-branch-name
