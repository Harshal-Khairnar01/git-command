What is git restore?

The git restore command is used to restore working directory files to a previous state. It can discard changes in files, restore them from the last commit, or even unstage files.

Basic Usage

1. Restore a single file to last committed version

git restore filename.txt

This replaces the file with its version from the last commit.

2. Restore multiple files

git restore file1.txt file2.txt

3. Restore an entire folder

git restore src/

4. Restore all changed files in the working directory

git restore .

Restoring staged changes

If you already staged changes with git add but want to unstage them:

git restore --staged filename.txt

Now the file remains modified in the working directory but is no longer staged for commit.

Restoring to a specific commit

git restore --source <commit-hash> filename.txt

This brings back the file as it was in the given commit.

Important Notes

  • Changes restored this way are lost if they are not committed anywhere.
  • Use with caution â€” there is no undo unless you have the changes saved in another commit or stash.
  • git restore was introduced in Git 2.23 (Aug 2019) to simplify commands like git checkout -- file.

Example Workflow

# You modify a file
echo "new content" >> hello.txt

# Check status
git status
# hello.txt is modified

# Discard changes
git restore hello.txt

# Now hello.txt is back to its last committed state
