Understanding git stash

The git stash command temporarily saves (or "stashes") your uncommitted changes in a safe place, so you can work on something else without losing your progress.

Why use Git Stash?

  • You want to switch branches, but have uncommitted changes.
  • You want to test something quickly without committing unfinished work.
  • You want to clean your working directory without deleting changes.

Basic Usage

git stash

This saves your uncommitted changes and clears them from your working directory.

View Stashes

git stash list

Shows all the stashes you've saved.

Apply a Stash

git stash apply

Applies the most recent stash without deleting it from the stash list.

Apply a Specific Stash

git stash apply stash@{2}

Replace 2 with the index of the stash you want to apply.

Pop a Stash

git stash pop

Applies the most recent stash and removes it from the stash list.

Drop a Stash

git stash drop stash@{0}

Deletes a specific stash without applying it.

Clear All Stashes

git stash clear

Removes all stored stashes.

Note: Stashes are stored locally and are not shared between repositories or branches.

Example Workflow

# You are working on feature-branch
git stash          # Save changes
git checkout main  # Switch to main branch
# ... do some work ...
git checkout feature-branch
git stash pop      # Get your changes back
