1. What is a Branch in Git?

A branch in Git is simply a pointer to a commit.

It allows you to work on different versions of your project independently.

The default branch name in Git is usually main or master.

Think of a branch as a separate workspace where you can make changes without affecting the main project.

If main branch has stable production code, you can create a feature/login branch to work on a login system without disturbing main.

2. Need for Branches

Branches are essential because they:

  • Enable parallel development â€“ Different team members can work on different features at the same time.
  • Avoid breaking main code â€“ New code is tested in a branch before merging.
  • Support experimentation â€“ Try out new ideas without affecting the working version.
  • Improve collaboration â€“ Each developer can push their branch and collaborate via Pull Requests (PRs).
  • Simplify bug fixing â€“ Create a bugfix/ branch to fix issues without touching other ongoing features.

Git Branch Basics

This page explains how to create, switch, merge, and delete branches in Git, with simple examples using the main branch.

1) Create a Branch

Create a new branch from your current branch (e.g., from main):

# From main
git checkout main
git pull

# Create branch (two options)
git branch feature-ui
# OR create and switch immediately:
git switch -c feature-ui
# (older syntax) git checkout -b feature-ui

Extra Examples:

  • git switch -c third â†’ Creates a new branch named third and switches to it in one step.
  • git checkout -b branch1 â†’ Older syntax that also creates a new branch named branch1 and switches to it.

Verify it exists:

git branch

2) Switch Between Branches

Switch to an existing branch:

git switch feature-ui
# (older syntax) git checkout feature-ui

Switch back to main:

git switch main
# (older syntax) git checkout main

3) Merge a Branch

Merge changes from a feature branch into main:

# Make sure you're on the target branch (where you want the result)
git switch main
git pull

# Merge the source branch into main
git merge feature-ui

Notes:

  • If Git can fast-forward, it will just move main forward.
  • If there are conflicts, Git will pause the merge so you can resolve files, then:
# After fixing conflicts in files:
git add .
git commit        # completes the merge (if not auto-committed)

Optional: create a merge commit even if fast-forward is possible:

git merge --no-ff feature-ui

4) Delete a Branch

Delete a local branch (after merging or if no longer needed):

# Safe delete (refuses if branch has unmerged commits)
git branch -d feature-ui

# Force delete (use with caution)
git branch -D feature-ui

Delete a remote branch on origin (GitHub/GitLab):

git push origin --delete feature-ui
# (alternative)
git push origin :feature-ui

Clean up stale remote-tracking branches locally:

git fetch --prune
# or
git fetch -p

Helpful Commands

# List local branches
git branch

# List remote branches
git branch -r

# List all (local + remote)
git branch -a

# Rename current branch
git branch -m new-name

# See current branch and status
git status

# Show a compact history graph
git log --oneline --graph --decorate --all

Suggested Workflow

# 1) Start from main
git switch main
git pull

# 2) Create and switch to feature branch
git switch -c feature-login

# 3) Work & commit on feature branch
git add .
git commit -m "Add login UI"

# 4) Merge back into main
git switch main
git pull
git merge feature-login

# 5) Push and delete the feature branch
git push
git branch -d feature-login
git push origin --delete feature-login   # optional

Tips

  • Always git pull latest changes into main before creating a feature branch.
  • Keep feature branches small and focused; merge back frequently.
  • Use meaningful branch names like feature/login, bugfix/payment-timeout, hotfix/production-crash.
  • Prefer git revert on shared branches if you need a safe undo without rewriting history.
